1. Prerequisites

System Requirements

  • 4+ CPU cores (recommended: 8 cores)
  • 8GB+ RAM (recommended: 16GB)
  • 100GB+ available disk space
  • Stable internet connection

Software Dependencies

Linux

# Ubuntu/Debian
sudo apt-get update
sudo apt-get install -y cmake ninja-build \
  libssl-dev libgtest-dev pkg-config \
  git build-essential

# CentOS/RHEL
sudo yum install -y cmake ninja-build \
  openssl-devel gtest-devel \
  pkgconfig git gcc-c++

macOS

# Install Homebrew if not installed
/bin/bash -c "$(curl -fsSL \
https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install dependencies
brew install cmake ninja openssl pkg-config

Windows

# Install Visual Studio 2019+ with C++ tools
# Install Git for Windows
# Install CMake from cmake.org

# Or use vcpkg (recommended)
git clone https://github.com/Microsoft/vcpkg.git
.\vcpkg\bootstrap-vcpkg.bat

2. Installation

Option 1: Build from Source (Recommended)

# Clone the repository
git clone https://github.com/r3e-network/neo_cpp.git
cd neo_cpp

# Initialize submodules
git submodule update --init --recursive

# Build using our professional script
./scripts/build.sh --type Release

# Verify build
ls -la build/apps/cli/neo-cli

Option 2: Docker (Quick Start)

# Pull the latest image
docker pull neo-cpp/node:latest

# Run with default configuration
docker run -d --name neo-node \
  -p 10333:10333 -p 20333:20333 \
  neo-cpp/node:latest

3. Configuration

Basic Configuration

Create a configuration file for your node:

{
  "ApplicationConfiguration": {
    "Logger": {
      "Path": "logs",
      "ConsoleOutput": true,
      "Active": true
    },
    "Storage": {
      "Engine": "LevelDBStore",
      "Path": "data"
    },
    "P2P": {
      "Port": 10333,
      "MinDesiredConnections": 10,
      "MaxConnections": 40
    },
    "RPC": {
      "BindAddress": "0.0.0.0",
      "Port": 20333,
      "SslCert": "",
      "SslCertPassword": "",
      "TrustedAuthorities": [],
      "MaxGasInvoke": 2000000000000,
      "MaxFee": 0,
      "MaxIteratorResultItems": 100,
      "SessionEnabled": false

  },
  "ProtocolConfiguration": {
    "Network": 860833102,
    "AddressVersion": 53,
    "MillisecondsPerBlock": 15000,
    "MaxTransactionsPerBlock": 512,
    "MemoryPoolMaxTransactions": 50000,
    "MaxTraceableBlocks": 2102400,
    "ValidatorsCount": 7,
    "CommitteeMembersCount": 21,
    "InitialGasDistribution": 5200000000000000,
    "Hardforks": {
      "HF_Aspidochelone": 1730000,

}

Configuration Tips

  • Use absolute paths for storage and logs
  • Adjust MaxConnections based on your bandwidth
  • Enable RPC only if you need API access
  • Configure firewall for ports 10333 and 20333

4. Running the Node

Start the Node

# Run with custom configuration
./build/apps/cli/neo-cli --config config.json

# Run in daemon mode (background)
./build/apps/cli/neo-cli --config config.json --daemon

# Run with verbose logging
./build/apps/cli/neo-cli --config config.json --verbose

Monitor Logs

# Follow real-time logs
tail -f logs/neo-cpp.log

# Monitor systemd service (if installed)
journalctl -u neo-cpp -f

# Check node status
curl http://localhost:20333/api/v1/status

5. Verification

Check Node Status

# Check if ports are open
netstat -tulpn | grep :10333
netstat -tulpn | grep :20333

# Test RPC connectivity
curl -X POST http://localhost:20333 \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"getversion","params":[],"id":1}'

# Check peer connections
curl -X POST http://localhost:20333 \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"getpeers","params":[],"id":1}'

Sync Status

Your node will start syncing with the Neo network. This may take several hours for the initial sync.

Sync Time

Initial blockchain sync can take 4-8 hours depending on your hardware and network speed. The node is fully operational once sync is complete.
