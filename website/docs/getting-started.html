Getting Started with Neo C++

Learn how to set up and run your first Neo node using the C++ implementation

Introduction

Neo C++ is a high-performance implementation of the Neo blockchain protocol written in modern C++. It provides enterprise-grade reliability, blazing-fast performance, and comprehensive tooling for building and deploying blockchain applications.

Why Neo C++?
  • 10,000+ TPS throughput capacity
  • Sub-50ms block processing time
  • Minimal memory footprint (<500MB)
  • 6,927 comprehensive tests
  • Production-ready with Docker and Kubernetes support

System Requirements

Minimum Requirements

  • CPU: 2 cores @ 2.0 GHz
  • RAM: 4 GB
  • Storage: 50 GB SSD
  • Network: 10 Mbps

Recommended Requirements

  • CPU: 4+ cores @ 3.0 GHz
  • RAM: 8+ GB
  • Storage: 100+ GB NVMe SSD
  • Network: 100+ Mbps

Supported Operating Systems

Linux

Ubuntu 20.04+, Debian 10+, RHEL 8+, Alpine 3.14+

macOS

macOS 11.0+ (Big Sur or later)

Windows

Windows 10/11, Windows Server 2019+

Any Docker-compatible platform

Installation

Docker Installation

The easiest way to get started is using Docker:

# Pull the official image
docker pull neocpp/node:latest

# Run a testnet node
docker run -d \
  --name neo-node \
  -p 10333:10333 \
  -p 10334:10334 \
  -v neo-data:/data \
  neocpp/node:latest --testnet

# Check node status
docker exec neo-node neo-cli state

# View logs
docker logs -f neo-node

Docker Compose

For production deployments, use Docker Compose:

# docker-compose.yml
version: '3.8'

services:
  neo-node:
    image: neocpp/node:latest
    container_name: neo-node
    restart: unless-stopped
    ports:
      - "10333:10333"  # P2P
      - "10334:10334"  # RPC
    volumes:
      - neo-data:/data
      - ./config:/config
    environment:
      - NEO_NETWORK=testnet
      - NEO_RPC_ENABLED=true
    command: --config /config/node.json

volumes:
  neo-data:

Building from Source

Build Neo C++ from source for maximum control:

# Clone the repository
git clone https://github.com/r3e-network/neo_cpp.git
cd neo-cpp

# Configure with CMake
cmake -B build \
  -DCMAKE_BUILD_TYPE=Release \
  -DBUILD_TESTS=ON \
  -DBUILD_SDK=ON

# Build (use all cores)
cmake --build build --parallel

# Run tests
cd build && ctest --parallel

# Install system-wide (optional)
sudo cmake --install build

Build Options

Option Default Description
CMAKE_BUILD_TYPE Release Build type (Debug, Release, RelWithDebInfo)
BUILD_TESTS OFF Build test suite
BUILD_SDK OFF Build SDK libraries
BUILD_PLUGINS ON Build plugin system
ENABLE_COVERAGE OFF Enable code coverage

Binary Releases

Download pre-built binaries for your platform:

# Linux (x86_64)
wget https://github.com/r3e-network/neo_cpp/releases/latest/download/neo-cpp-linux-x64.tar.gz
tar -xzf neo-cpp-linux-x64.tar.gz
cd neo-cpp
./neo_node --testnet

# macOS (Apple Silicon)
wget https://github.com/r3e-network/neo_cpp/releases/latest/download/neo-cpp-darwin-arm64.tar.gz
tar -xzf neo-cpp-darwin-arm64.tar.gz
cd neo-cpp
./neo_node --testnet

# Windows
# Download neo-cpp-windows-x64.zip from releases
# Extract and run neo_node.exe
Security Note: Always verify checksums of downloaded binaries: 
sha256sum neo-cpp-linux-x64.tar.gz

Package Managers

Homebrew (macOS/Linux)

# Add Neo tap
brew tap neo-project/neo

# Install Neo C++
brew install neo-cpp

# Run node
neo-node --testnet

APT (Ubuntu/Debian)

# Add Neo repository
curl -fsSL https://apt.neo.org/gpg | sudo gpg --dearmor -o /usr/share/keyrings/neo-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/neo-archive-keyring.gpg] https://apt.neo.org stable main" | sudo tee /etc/apt/sources.list.d/neo.list

# Install
sudo apt update
sudo apt install neo-cpp

# Run as service
sudo systemctl start neo-node
sudo systemctl enable neo-node

Configuration

Neo C++ uses JSON configuration files. Create a config.json:

{
  "ApplicationConfiguration": {
    "Network": "testnet",
    "Storage": {
      "Engine": "RocksDB",
      "Path": "./data"
    },
    "P2P": {
      "Port": 10333,
      "MaxConnections": 10,
      "MinDesiredConnections": 5
    },
    "RPC": {
      "Enabled": true,
      "Port": 10334,
      "MaxConcurrentConnections": 40,
      "MaxGasInvoke": 20
    },
    "UnlockWallet": {
      "Path": "./wallet.json",
      "Password": "",
      "IsActive": false
    },
    "Plugins": [
      "ApplicationLogs",
      "RpcServer",
      "StateService"
    ]
  }
}

Network P2P Port RPC Port Description
mainnet 10333 10331 Production network
testnet 20333 20331 Test network
private 30333 30331 Private network

Running Your First Node

1. Start the Node

# Run with default testnet configuration
./neo_node --testnet

# Or with custom config
./neo_node --config config.json

# Run in background
nohup ./neo_node --testnet > neo.log 2>&1 &

2. Check Node Status

# Using neo-cli
./neo-cli state

# Using RPC
curl http://localhost:10334 \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"getblockcount","params":[],"id":1}'

3. Monitor Synchronization

# Watch block height
watch -n 2 "./neo-cli getblockcount"

# View peers
./neo-cli show peers

# Check memory pool
./neo-cli show pool
Congratulations! You've successfully set up your first Neo C++ node. Next, explore the SDK Guide to start building applications.
